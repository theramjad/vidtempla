Key Design Decisions

SELECT FOR UPDATE locking

Each video is row-locked inside a transaction before comparing descriptions. This prevents race conditions when multiple sync jobs run concurrently — only one can detect drift for a given video at a time.

Deduplication

If the latest description_history entry already matches the live description and is marked manual_youtube_edit, we skip inserting a duplicate. We still flag driftDetectedAt to ensure the gate catches it.

Baseline on first sync

When a channel has never been baselined (driftBaselinedAt IS NULL), the sync silently updates currentDescription without flagging drift. This prevents false positives on every existing video when drift detection is first deployed.

Unmanaged videos pass through

Videos without a container (containerId = null) aren't managed by the template system, so description changes are silently accepted without flagging drift.

Surfaces

  • Dashboard — Drift badge on VideosTab, banner + resolve menu in HistoryDrawer, confirm dialogs on EditVariablesSheet and EditContainerModal
  • MCP — check_drift, resolve_drift tools; force param on all write tools
  • REST API — /videos/:id/check-drift, /videos/:id/resolve-drift endpoints; force query param on push endpoints
