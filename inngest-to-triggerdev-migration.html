VidTempla: Inngest to Trigger.dev Migration

Background job architecture before and after migration

Before: Inngest Architecture

sequenceDiagram participant UI as Dashboard / API participant Router as tRPC Router
youtube.ts participant Inngest as Inngest Client
inngestClient.send() participant API as /pages/api/inngest.ts
serve() endpoint participant Cloud as Inngest Cloud participant Fn as Inngest Functions
/inngest/youtube/*.ts Note over UI, Fn: Event-driven: call sites fire named events, Inngest routes them UI->>Router: User action (sync, update, etc.) Router->>Inngest: inngestClient.send({ name, data }) Inngest->>Cloud: POST event to Inngest Cloud Cloud->>API: Webhook callback to /api/inngest API->>Fn: Route to matching function Fn->>Fn: Execute steps (step.run) Note over Cloud, API: Inngest Cloud manages
scheduling, retries, cron

After: Trigger.dev Architecture

sequenceDiagram participant UI as Dashboard / API participant Router as tRPC Router
youtube.ts participant SDK as Trigger.dev SDK
tasks.trigger() participant Cloud as Trigger.dev Cloud participant Task as Trigger.dev Tasks
/src/trigger/*.ts Note over UI, Task: Direct task triggering: call sites invoke tasks by ID UI->>Router: User action (sync, update, etc.) Router->>SDK: tasks.trigger("task-id", payload) SDK->>Cloud: Trigger task via API Cloud->>Task: Execute task in managed worker Task->>Task: Run task logic directly Note over Cloud, Task: Trigger.dev Cloud manages
scheduling, retries, cron, concurrency

Key Concept Mapping

Inngest ConceptTrigger.dev EquivalentNotes
inngestClient.send({ name, data }) tasks.trigger("task-id", payload) Direct task invocation replaces named events
inngestClient.createFunction() task({ id, run }) Simpler API, no step.run() wrappers needed
{ cron: '...' } trigger schedules.task({ cron: '...' }) Dedicated scheduled task type
NonRetriableError AbortTaskRunError Same purpose: skip retries for known-bad state
concurrency: { limit: 5 } queue: { concurrencyLimit: 5 } Queue-based concurrency control
onFailure callback onFailure callback Same pattern, slightly different payload shape
serve() API route No API route needed Trigger.dev uses its own managed worker
